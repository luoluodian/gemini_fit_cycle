## 7.3 手动验证计划 (组件单元测试)

### 用例 1: BaseButton 样式验证 (Visual Check)
- **操作**: 在开发页面中依次渲染 `type="primary"`, `type="secondary"`, `type="wechat"`, `type="danger"`。
- **预期**:
  - `primary`: 翡翠绿渐变背景，白色文字。
  - `secondary`: 灰色边框，白色背景，灰色文字。
  - `wechat`: 微信绿渐变背景，包含 SVG 图标。
  - `danger`: 红色渐变背景。

### 用例 2: 点击反馈验证 (Interaction)
- **操作**: 点击任意按钮。
- **预期**: 按钮瞬间缩小 (`scale-95`)，松开后恢复，提供明确的触感反馈。

### 用例 3: 加载状态验证 (Loading State)
- **操作**: 为 `BaseButton` 传入 `:loading="true"`。
- **预期**:
  - 按钮中心显示旋转的 Spinner 动画。
  - 按钮文字/图标隐藏或置灰。
  - 点击按钮不触发 `@click` 事件。

### 用例 4: 事件派发验证 (Events)
- **操作**: 点击 `LoginButton` 中的两个按钮。
- **预期**:
  - 点击“微信一键登录”触发父组件的 `@login` 事件。
  - 点击“先体验一下”触发父组件的 `@guest` 事件。
