# F-1: 食物列表接口 - T5 分析文档

## 5.1 需求说明
- **目标**: 为前端提供可滚动的食物基础库列表。
- **关键点**: 分页 (Pagination) 与 排序 (Sorting)。

## 5.2 技术实现点
- **分页**: 使用 `offset` 和 `limit` (通过 `applyPagination` 封装)。
- **排序**: 支持按热量、蛋白质等字段进行升降序排列。
- **稳定性**: 强制设定默认每页数量，防止拉取全表。

## 5.3 验收标准
- [ ] 能按页码返回数据。
- [ ] 返回结果包含 `total` 总数以便前端计算分页。
- [ ] 默认按 ID 降序排列。
