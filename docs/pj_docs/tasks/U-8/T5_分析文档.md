# U-8: 档案展示组件 - T5 分析文档

## 5.1 模块关联性审计

- **涉及公共组件**:
  - `BaseButton.vue`: 统一按钮样式。
  - `GlassCard.vue`: 统一毛玻璃卡片容器。
- **业务组件审计**:
  - `QuickStats.vue`: 统计数据展示。
  - `HealthTools.vue`: 健康工具列表。
  - `SettingsMenu.vue`: 设置菜单列表。
  - `CurrentPlanCard.vue`: 当前计划摘要。
  - `ImportPlanCard.vue`: 导入入口。

## 5.2 影响评估

- **代码冗余**: 菜单项 (`menu-item`) 的样式和结构在多个组件中重复。
- **视觉不统一**: 部分组件未完全应用 U-4 交付的 `BaseButton`。
- **扩展性**: 缺乏统一的菜单项配置模式，后续新增功能需手动编写大量 HTML。

## 5.3 技术落地计划

### 1. 深度原子化
- **提取 `GlassCard`**: 确保全站卡片边框、阴影、模糊度一致。
- **重构 `MenuItem`**: 封装左侧图标、中间标题/描述、右侧箭头及点击态。

### 2. 组件抛光
- **按钮接入**: 将 `ProfileHeader`, `CurrentPlanCard`, `ImportPlanCard` 中的原生按钮替换为 `BaseButton`。
- **状态联动**: 统一处理点击时的交互反馈 (`active:scale-95` 或 `translateX`)。

### 3. 文档化
- 完善各组件的 Props 定义，支持更灵活的内容注入。

## 5.4 验收标准 (T7 预案)
- [ ] 个人中心所有卡片视觉风格高度一致（间距、圆角、模糊度）。
- [ ] 菜单项点击时有统一的背景变色或位移反馈。
- [ ] 所有按钮调用均基于 `BaseButton`。
- [ ] 代码量显著减少（通过组件复用）。
