# F-3: 食物详情接口 - T5 分析文档

## 5.1 模块关联性审计
- **核心功能**: 根据 ID 获取单个食物的所有营养指标（热量、三大营养素、GI 等）。
- **关联组件**: 前端食物详情弹窗 (`FoodDetailModal.vue`)。

## 5.2 接口设计
- **路径**: `GET /food/:id`
- **校验**: ID 必须为正整数。
- **数据完整性**: 必须返回 `foods` 表中除逻辑删除位外的所有业务字段。

## 5.3 实现逻辑
1. Controller 接收 `:id` 路径参数。
2. Service 调用 `this.foodRepo.findOne({ where: { id } })`。
3. 若为空，返回 `404` 或包含 error 的对象。

## 5.4 验收标准
- [ ] 能通过正确 ID 获取完整对象。
- [ ] 访问不存在 ID 返回错误提示。
