## 2. 核心测试用例 (Core Test Cases)

### 2.1 后端数据同步 (Backend Data Sync)
| 步骤 ID | 测试动作 | 预期结果 |
| :--- | :--- | :--- |
| **DS-1** | 发送 `POST /api/food-items/sync` 触发同步 | 接口返回 `count` 数量且状态码 201 |
| **DS-2** | 检查数据库 `food_items` 表 | `type='system'` 的记录与 `html/food.js` 数据完全对齐 |
| **DS-3** | 重复触发同步接口 | 数据库执行 Upsert 或覆盖，无重复数据产生 |

### 2.2 食材发现与搜索 (Discovery & Search)
| 步骤 ID | 测试动作 | 预期结果 |
| :--- | :--- | :--- |
| **SR-1** | 首页搜索框输入“鸡蛋” | 列表实时刷新，显示名称包含“鸡蛋”的所有食材 |
| **SR-2** | 切换分类（如：肉类、蛋类） | 列表仅显示对应 `category` 的食材 |
| **SR-3** | 组合查询：输入名称 + 切换分类 | 结果集为两者的交集 |
| **SR-4** | 分页加载：向下滑动列表到底部 | 触发 `searchFoodItems` 下一页请求，数据无缝追加 |

### 2.3 自定义食材 CRUD 与查重 (Custom Food & Similarity)
| 步骤 ID | 测试动作 | 预期结果 |
| :--- | :--- | :--- |
| **CF-1** | 点击创建按钮，输入已存在的系统食材名称 | 系统提示“已存在类似食材”，防止重复创建 |
| **CF-2** | 完整填写名称、单位(g/ml)、营养素并保存 | 接口返回成功，列表刷新出“个人”标识的食材 |
| **CF-3** | 修改个人食材的单位或热量值 | 更新成功，详情页显示最新数值 |
| **CF-4** | 删除已创建的个人食材 | 食材从列表中消失，后端 `food_items` 记录物理或逻辑删除 |

### 2.4 收藏体系验证 (Favorites Logic)
| 步骤 ID | 测试动作 | 预期结果 |
| :--- | :--- | :--- |
| **FV-1** | 点击食材卡片上的“收藏”图标 | 图标变色，触发 `POST /favorite` 接口 |
| **FV-2** | 在“已收藏”分类中查看 | 刚才收藏的食材出现在该分类顶部 |
| **FV-3** | 取消收藏并刷新页面 | 该食材不再出现在收藏列表中，状态同步至后端 |

### 2.5 权限隔离与安全性 (Permissions & Security)
| 步骤 ID | 测试动作 | 预期结果 |
| :--- | :--- | :--- |
| **PS-1** | 登录账号 A，创建私有食材 | 退出并登录账号 B |
| **PS-2** | 账号 B 搜索账号 A 的私有食材 | 搜索结果为空，确保数据纵向隔离 |
| **PS-3** | 尝试通过 ID 直接请求 A 的私有食材详情 | 后端应返回 403 或 404，禁止越权访问 |

### 2.6 单位与业务准确性 (Units & Accuracy)
| 步骤 ID | 测试动作 | 预期结果 |
| :--- | :--- | :--- |
| **UA-1** | 检查“按个”计量的食材（如：中等鸡蛋） | 热量基准应按“1个”计算而非“100g” |
| **UA-2** | 在创建页面选择单位“ml” | 确认保存后，数据库中 `unit` 字段记录为 `ml` |

### 2.7 极端值与输入校验 (Extreme Values & Validation)
| 步骤 ID | 测试动作 | 预期结果 |
| :--- | :--- | :--- |
| **EV-1** | 在热量、蛋白质等数值字段输入负数 (e.g., -100) | 前端拦截或后端 `ValidationPipe` 报错，禁止保存 |
| **EV-2** | 输入极大数值 (e.g., 999999) | 验证 UI 布局（卡片、详情页）是否因数字过长导致折行或溢出 |
| **EV-3** | 食材名称输入超长字符串 (e.g., > 100 字符) | 系统应限制输入长度或按数据库字段定义截断/报错 |
| **EV-4** | 食材名称包含特殊字符、Emoji 或 SQL 敏感词 | 系统应正确转义并显示，不得触发 500 错误或注入风险 |
| **EV-5** | 提交必填项为空的自定义食材 | 触发 `class-validator` 校验，提示具体缺失字段 |

## 3. 回归测试重点 (Regression Focus)
- 确保食材库的任何修改不影响 **饮食记录 (Diet Records)** 模块的食材检索速度。
- 确认 **搜索防抖 (Debounce)** 逻辑正常，避免短时间内产生大量重复请求。
