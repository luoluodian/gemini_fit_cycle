# Food 模块测试用例设计 (F-Module Test Cases)

## 1. 测试目标
验证食材库模块的核心功能，包括搜索、筛选、收藏、自定义食材管理以及营养素详情展示。

## 2. 测试场景与用例

### 2.1 搜索与筛选 (Search & Filter)
| ID | 测试场景 | 预期结果 | 优先级 |
|:---|:---|:---|:---|
| TC-F-001 | 关键词搜索 | 输入“鸡”，列表应显示“鸡胸肉”等匹配项。 | P0 |
| TC-F-002 | 分类切换 | 点击“蛋白质”分类，列表应仅显示该分类下的食材。 | P0 |
| TC-F-003 | 无结果处理 | 搜索不存在的食材，应显示“暂无食物”缺省页。 | P1 |
| TC-F-004 | 搜索重置 | 清空搜索框后，应恢复显示全部或默认列表。 | P1 |

### 2.2 食材详情 (Food Detail)
| ID | 测试场景 | 预期结果 | 优先级 |
|:---|:---|:---|:---|
| TC-F-010 | 详情页展示 | 点击食材卡片，弹出 Modal，显示 Emoji、分类、热量及三大营养素。 | P0 |
| TC-F-011 | 数量步进器联动 | 在详情页修改摄入克数，营养素数值应实时按比例重算。 | P0 |
| TC-F-012 | 快捷克数切换 | 点击 50g/100g 等快捷选项，克数及数值应正确更新。 | P1 |

### 2.3 收藏功能 (Favorites)
| ID | 测试场景 | 预期结果 | 优先级 |
|:---|:---|:---|:---|
| TC-F-020 | 点击收藏 | 在详情页点击心形图标，图标变为实心红色，且数据持久化成功。 | P0 |
| TC-F-021 | 取消收藏 | 再次点击已收藏食材，图标恢复空心，状态更新。 | P0 |
| TC-F-022 | 列表状态联动 | 列表中的收藏状态应与详情页及后端保持一致。 | P1 |

### 2.4 自定义食材 (Custom Foods)
| ID | 测试场景 | 预期结果 | 优先级 |
|:---|:---|:---|:---|
| TC-F-030 | 创建自定义食材 | 输入名称、热量、营养素，保存后列表中应出现该食材。 | P0 |
| TC-F-031 | 字段验证 | 名称为空或数值非法时，提交应有错误提示。 | P1 |
| TC-F-032 | 修改自定义食材 | 仅创建者可修改自己的食材，保存后更新。 | P0 |
| TC-F-033 | 删除自定义食材 | 删除后列表中消失。 | P0 |
| TC-F-034 | 权限隔离 | 系统食材不应出现“编辑/删除”按钮。 | P1 |

### 2.5 热门食材 (Popular Foods)
| ID | 测试场景 | 预期结果 | 优先级 |
|:---|:---|:---|:---|
| TC-F-040 | 热门列表展示 | 首页或食材库推荐区域应显示收藏量最高的食材。 | P1 |
