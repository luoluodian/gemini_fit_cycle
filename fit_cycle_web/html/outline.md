# 饮食记录H5应用项目概述

## 项目简介
一个功能完整的移动端饮食记录与管理H5应用，帮助用户科学管理日常饮食、制定个性化饮食计划、追踪营养摄入。

## 技术架构
- **前端框架**：原生HTML5 + CSS3 + JavaScript
- **样式系统**：Tailwind CSS
- **动画库**：Anime.js, ECharts.js, Splide.js
- **数据存储**：LocalStorage (客户端存储)
- **响应式设计**：移动优先设计

## 页面结构

### 1. 记录(今日)页面 (index.html)
**功能定位**：应用核心页面，用户日常使用的主界面

**主要内容**：
- 顶部日期导航区（今日日期、左右切换、计划提示）
- 今日目标概览（热量、蛋白质、脂肪、碳水目标）
- 餐次记录区（早餐、午餐、晚餐、加餐）
- 每日汇总统计（总摄入、目标对比、建议反馈）

**交互功能**：
- 日期切换和选择
- 添加/编辑食物记录
- 按计划快速记录
- 实时统计更新

### 2. 计划管理页面 (plan.html)
**功能定位**：饮食计划的创建、管理和配置中心

**主要内容**：
- 计划列表展示（进行中/已归档）
- 计划创建和编辑表单
- 每日目标配置界面
- 餐单配置功能
- 计划分享和导入

**交互功能**：
- 计划状态管理（激活/取消激活）
- 多步骤表单填写
- 日历选择和配置
- 分享码生成和解析

### 3. 食材库页面 (food.html)
**功能定位**：食材信息管理和搜索中心

**主要内容**：
- 食材搜索和筛选
- 系统食材和自定义食材分类
- 食材详细信息展示
- 最近使用和常用食材
- 自定义食材创建

**交互功能**：
- 实时搜索和筛选
- 食材详情查看
- 自定义食材编辑
- 常用食材标记

### 4. 个人中心页面 (profile.html)
**功能定位**：用户信息管理和工具集合

**主要内容**：
- 用户基础信息展示
- 当前激活计划状态
- 目标建议工具（BMR/TDEE计算）
- 周统计图表
- 设置和工具入口

**交互功能**：
- 个人信息编辑
- 健康数据计算
- 统计数据可视化
- 计划导入功能
