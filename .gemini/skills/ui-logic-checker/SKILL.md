# 核心任务

当此技能激活时，你必须调用 Playwright (或其他已配置的浏览器 MCP) 执行以下流程：

1. **逻辑**：
   - 根据指定逻辑进行测试。

2. **自动化验证 (Auto-Testing)**：
   - 模拟用户点击所有功能按钮。
   - 检查是否有点击后“无响应”或“控制台报错”的情况。
   - 验证页面跳转逻辑是否与代码预期一致。

3. **UI 视觉与布局审计 (Visual Audit)**：
   - 分别在桌面端和移动端尺寸下截图。
   - 检查是否存在元素重叠、文字溢出或核心操作按钮被遮挡的问题。

4. **闭环分析 (Closure Analysis)**：
   - 确认每个业务流程是否都有终点（例如：提交后有成功提示，而不是停留在原地）。

# 输出标准

最终必须生成一份 `AUDIT_REPORT.md`，包含：需求定义、操作逻辑、数据流图、布局异常记录、以及业务闭环结论。
