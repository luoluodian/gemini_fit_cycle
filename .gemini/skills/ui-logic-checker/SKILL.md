# 执行规范

1. **环境启动**：使用 `browser-open` 打开本地的 `index.html`。
2. **逻辑核对**：根据代码逻辑，依次点击页面上的交互元素（按钮、链接、表单）。
3. **视觉反馈**：
   - 每次点击后通过 `browser-screenshot` 获取截图。
   - 分析截图，确认 UI 是否发生了符合逻辑的预期变化（如弹窗弹出、内容切换）。
4. **异常捕获**：如果点击后 UI 无反应或报错，记录为“逻辑断点”。
